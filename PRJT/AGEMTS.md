1. 프로젝트 개요

프로젝트명: 소상공인 1초 세금비서 (가칭: 세복이)
개발 기간: 4일 (데모 시연 목적의 초고속 MVP 개발)
핵심 타깃: 4060 중장년 소상공인 (개인택시, 식당, 카페, 도소매업 사장님)
기획 배경: 바쁜 소상공인들이 세무 장부 작성(CSV 다운로드 및 업로드 등)에 겪는 어려움을 해결하기 위해, 원클릭 모의 연동과 영수증 촬영으로 지출을 자동 분류해 주는 직관적인 AI 웹 대시보드 제공.
메인 캐릭터: '세복이' (복을 가져다주고 세금을 아껴주는 귀여운 3D 복주머니 요정)


2. 아키텍처 및 기술 스택

프론트엔드 (UI): React (PC 웹 브라우저 시연용 16:9 가로형 레이아웃)
백엔드 (API): FastAPI, python, java, js, React, HTML, CSS
데이터베이스: SQLite

AI 및 멀티에이전트 (OpenAI API 활용)
    Data Agent: 업로드된 데이터(모의 영수증, 텍스트) 파싱 및 DB 적재
    Tax Expert Agent: LLM 프롬프트를 통해 지출 내역을 분석하고 '사업용(절세)'과 '개인용'으로 자동 분류

구성은 폴더 구조로 생성, 예를 들어 html, js, css 등 각 소스별로 폴더로 구성해서 진행
시스템 구성하고 프로그램 개발 시 최대한 알기 쉽게 복잡한 부분을 간소화하고 이해하기 쉽게 구축을 진행한다

3. UI/UX 디자인 워크플로우 (Canva + Stitch)

본 프로젝트는 속도와 퀄리티를 위해 아래의 툴 연계 방식을 사용합니다.

Canva (재료 준비): 3D 아이콘(열쇠, 카메라) 및 세복이 캐릭터를 디자인하고, 반드시 배경이 투명한 PNG 파일로 추출합니다.
Stitch (조립 및 코드 변환): Canva에서 추출한 투명 에셋을 Stitch에 업로드하여 최종 화면을 조립하고, React 소스 코드로 내보내기 합니다.
색상 관련은 DESIGN.md 참조해서 진행한다
UI 생성 시 ./html/smp.html 파일을 react로 변환해서 화면 UI를 개발한다

디자인 핵심 규칙
    영어를 완전히 배제하고 직관적인 한글 사용.
    침침한 눈에도 잘 보이도록 폰트 크기 20% 확대 및 높은 대비 적용.
    뻔한 차트 대신 캐릭터와 3D 아이콘을 활용하여 피로감 감소.

5. 2~4일 완성 데모(시연) 시나리오

실제 은행 API 연동 없이도 완벽한 시연을 보여주기 위한 프론트엔드 모의(Mock) 동선입니다.
Scene 1 (첫인상): 웹 브라우저 실행 시 세복이와 큼직한 한글 대시보드가 노출됩니다.
Scene 2 (자동 연동 클릭): '내 계좌 연결하기' 클릭 시 ➡️ 2초간 로딩 애니메이션 ➡️ "🎉 연동 완료! 숨은 절세 항목 14건을 찾았습니다." 팝업 노출 및 대시보드 수치 변화.
Scene 3 (영수증 촬영 클릭): '영수증 촬영하기' 클릭 시 ➡️ 가상의 영수증 이미지 업로드 ➡️ FastAPI 서버 연동 모의 분석 ➡️ "식재료비 5만원 - 사업용 지출로 분류 완료" 알림 발생.


6. 팀 협업 및 깃허브(GitHub) 규칙

보안을 위해 반드시 프라이빗(Private) 레포지토리를 생성하여 팀원을 Collaborator로 초대합니다.
